--- title: Transcodeur d'images ---

Utilisation du service de transcodage d'images App Cloud, qui peut grandement améliorer les performances de votre application en optimisant les images pour la diffusion mobile.

App Cloud fournit un service que vous pouvez utiliser pour transcoder des images pour vos applications. En utilisant le service de transcodage d'images, vous pouvez prendre votre image source et la convertir dans la taille et le format dont vous avez besoin pour vos applications. Par exemple, si vous avez une image originale de 300 x 300 pixels, mais que votre application doit l'afficher en 100 x 100. Vous pouvez utiliser le service de transcodage d'image pour vous assurer que l'image est toujours à la bonne taille pour son contexte, sans avoir à créer un nouveau fichier image pour chaque contexte dans lequel vous utilisez l'image. Vous pouvez également recadrer l'image et spécifier la qualité (taux de compression) des images JPG. En utilisant le service de transcodage d'images, vous pouvez :

  • Améliorez considérablement les performances de votre application en réduisant considérablement la taille du fichier et la bande passante réseau requise pour afficher votre application
  • Réduisez les besoins en mémoire en réduisant la taille des images.
  • Assurez-vous que vos images sont de la bonne taille pour être affichées sur les appareils mobiles.
  • Tirez parti du cache du serveur App Cloud, améliorant les performances des applications et réduisant la charge sur vos propres serveurs.

Le service de transcodage d'images App Cloud agit comme un proxy entre vos images et vos applications. Le service est accessible avec une simple URL dans laquelle vous préfixez l'URL de votre image avec l'URL de notre service de transcodage. Cela vous permet de déposer notre service de transcodage dans vos applications avec très peu d'effort. Par exemple, si vous avez une image qui vit à http://mon.exemple.com/image.png vous le demanderiez normalement comme :

  <img  src="http://my.example.com/image.png"  alt="Image affamée de bande passante" width="500" height="500" />

Pour le transcoder en une image plus petite de 100x100px, il vous suffit d'émettre la requête suivante :

  <img  src="http://transcode.appcloud.brightcove.com?image=http://my.example.com/image.png&width=100&height=100"  alt="Mon image svelte" width="100" height="100" />

Pour réduire une image à 100px de largeur, en préservant le rapport hauteur/largeur d'origine :

  <img  src="http://transcode.appcloud.brightcove.com?image=http://my.example.com/image.png&width=100" width="100" />

Pour réduire une image à 100 px de largeur ou 100 px de hauteur, selon la valeur la plus élevée, en préservant le rapport hauteur/largeur d'origine :

  <img  src="http://transcode.appcloud.brightcove.com?image=http://my.example.com/image.png&max_dimension=100" />

Pour recadrer l'image, en conservant une pièce de 100x100px au centre, il vous suffit d'émettre la requête suivante :

  <img  src="http://transcode.appcloud.brightcove.com?image=http://my.example.com/image.png&crop=200,200,100,100"  alt="Mon image svelte" width="100" height="100" />

Pour utiliser le service de transcodage d'images, votre image doit être accessible à une URL accessible au public. L'image doit être au format PNG, JPEG ou GIF.

Paramètres de requête

L'appel de service de transcodage d'image utilise les paramètres de requête suivants :

image (obligatoire)

L'URL accessible au public à laquelle votre image originale peut être trouvée. Par exemple, image=http://my.example.com/image.png.

format (facultatif)

Le format de la nouvelle image, qui peut être l'un des png , jpg , ou gif. Si vous ne spécifiez pas le format, le format de sortie est inchangé par rapport à l'image d'origine.

Paramètres de taille (facultatif)

Vous pouvez utiliser l'un des paramètres de taille, largeur , la taille , ou dimension_max , si vous souhaitez que la nouvelle image soit d'une taille différente de l'image d'origine. Ces paramètres spécifient une dimension de l'image finale en pixels.

  • Si vous ne spécifiez aucun paramètre de taille, la nouvelle image aura la même largeur et la même hauteur que l'image d'origine.
  • Si vous spécifiez largeur , mais non la taille , la nouvelle image aura la largeur que vous spécifiez et la hauteur dictée par le rapport hauteur/largeur de l'image d'origine.
  • Si vous spécifiez la taille , mais non largeur , la nouvelle image aura la hauteur que vous spécifiez et la largeur dictée par le rapport hauteur/largeur de l'image d'origine.
  • Si vous spécifiez dimension_max , alors la plus grande dimension de la nouvelle image sera la taille que vous spécifiez, et la plus petite dimension aura la taille dictée par le rapport hauteur/largeur de l'image d'origine.

Dans tous les cas, la nouvelle image aura le même rapport hauteur/largeur que l'image d'origine, sauf si vous spécifiez à la fois un largeur et la taille et ce rapport hauteur/largeur est différent du rapport hauteur/largeur d'origine.

Paramètre de recadrage (facultatif)

Utilisation d'une fonction utilitaire de transcodage

Pour recadrer votre image, ajoutez le paramètre :

recadrer=x,y,w,h

Ici, x et y sont les coordonnées du point de départ, mesurées en pixels à partir du coin supérieur gauche (0,0). w et h sont des pixels horizontaux et verticaux à préserver, en commençant au point de départ et en continuant vers la droite et le bas.

Si le recadrage et le dimensionnement sont tous deux spécifiés, le recadrage est appliqué en premier, puis l'image recadrée est redimensionnée.

Paramètre de qualité (facultatif)

Pour les images JPG uniquement, spécifiez le taux de compression de l'image résultante en ajoutant le paramètre :

qualité=x

x est le taux de compression à appliquer ; la valeur par défaut est 70.

Voici une fonction JavaScript pratique pour générer une URL de transcodage d'image :

  fonction getTranscodeURL(url, options) { var src = "http://transcode.appcloud.brightcove.com?image=" + url; if (options) { if (options.max_dimension) { src += "&max_dimension=" + options.max; } autre { if (options.width) { src += "&width=" + options.width; } si (options.hauteur) { src += "&height=" + options.height; } } si (options.format) { src += "&format=" + options.format; } si (options.crop) { src += "&crop=" + options.crop; } si (options.qualité) { src += "&quality" + options.quality; } } renvoie src; } ;

Par exemple, pour transcoder une image en 200 pixels de large :

  var tx_url = getTranscodeURL("http://www.example.com/example.jpg", { width: 200 });

Pour recadrer une image à 160 x 90, en commençant à 100 pixels à partir de la gauche et à 70 pixels en partant du haut, et en spécifiant un taux de compression de 65 :

var tx_url = getTranscodeURL("http://www.example.com/example.jpg", { crop: "100,70,160,90" , quality: 65 });
